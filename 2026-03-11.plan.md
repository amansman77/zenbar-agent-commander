# Zenbar Agent Commander

Personal Remote Codex Orchestration Control Plane

## Glossary

| Term                   | Meaning                     |
| ---------------------- | --------------------------- |
| Zenbar Agent Commander | 전체 시스템                |
| Web Commander          | 웹 UI                      |
| Mobile Commander       | iPhone app                 |
| Orchestration API      | Zenbar backend             |
| Codex App Server       | agent runtime              |
| Codex CLI              | local coding engine        |
| Task Workspace         | task isolation environment |

## Goal

Build a personal orchestration control plane that allows developers to supervise AI coding agents from anywhere.

Zenbar does not implement its own agent runtime. It coordinates task lifecycle, visibility, and approval above the Codex App Server runtime.

## Canonical Architecture

Zenbar Agent Commander runs as a control plane above the Codex App Server runtime.

```text
Web Commander / Mobile Commander
            ↓
Zenbar Orchestration API
            ↓
       Codex App Server
            ↓
        Codex Runtime
            ↓
Codex CLI / tools / filesystem / git
```

## Agent Runtime

Agent execution is handled by Codex App Server.

Zenbar does not implement its own agent runtime.

The Codex App Server owns:

* agent session lifecycle
* runtime state
* agent execution control

Zenbar owns:

* task lifecycle
* approval state
* project and task visibility
* remote supervision interfaces

## Product Strategy

Development follows a web-first validation approach.

Phase 1 validates the orchestration workflow through Web Commander. Phase 2 packages the same interface as Mobile Commander with Capacitor.

## System Components

### Web Commander

Primary control interface for:

* task creation

* event viewing
* diff review
* approval actions

### Mobile Commander

Mobile interface packaged from the same web application using Capacitor.

Primary responsibilities:

* remote monitoring
* approval control

### Orchestration API

The Zenbar backend manages task lifecycle and exposes control-plane APIs.

Responsibilities:

* project management
* task creation
* task state transitions
* event ingestion
* approval records
* persistence

### Codex App Server

The Codex App Server provides the agent runtime.

Responsibilities:

* agent session lifecycle
* runtime execution
* agent state management

### Codex CLI and Local Tools

The runtime executes code work through Codex CLI and host-local tools.

Local tools include:

* git
* shell
* test commands
* docker when needed

## Approval Semantics

Codex agents operate inside an isolated task workspace.

Human approval is required before a task result is accepted as final.

This policy means:

* agents may freely modify files in the task workspace
* pre-approval state remains isolated
* accepted results require explicit approval

## Task Workspace

Each task runs inside an isolated workspace (branch or worktree).

Branch format:

```text
task/<slug>-<shortid>
```

Examples:

```text
task/fix-canonical-a1b2
task/add-dashboard-k3x1
```

Isolation strategies:

* Option A: git branch
* Option B: git worktree

## Agent Execution Flow

```text
Create Task
↓
Create Task Workspace
↓
Start runtime via Codex App Server
↓
Agent modifies isolated workspace
↓
Stream execution events to Zenbar
↓
Review diff and task status
↓
Human approval
↓
Accepted result
```


## MVP Scope

The MVP is successful if the following workflow works:

1. Web Commander creates a task.
2. Orchestration API tracks task lifecycle.
3. Codex App Server executes the agent runtime.
4. execution events stream to the UI.
5. diff review is visible.
6. user approves the task result.

## Guiding Principle

Zenbar is not a mobile IDE and not a replacement for the Codex runtime.

Zenbar is a remote orchestration console for supervising AI coding agents.

## Summary

Zenbar Agent Commander is a control plane above the Codex App Server runtime.

Its value comes from remote supervision, human approval, multi-project orchestration, and mobile control.
