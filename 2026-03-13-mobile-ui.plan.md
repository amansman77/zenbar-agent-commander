# 2026-03-13 Mobile UI Refactor Plan (Zenbar Agent Commander)

## Objective

현재 Zenbar Web Commander UI는 데스크톱 3‑panel 레이아웃(Projects / Tasks / Task Detail)에 최적화되어 있으며 모바일 환경에서는 다음과 같은 UX 문제가 발생한다.

* 패널이 세로로 누적되어 정보 계층이 불명확해짐
* Plan output 영역에서 좌우 스크롤 발생
* Task Detail이 너무 긴 스크롤 영역이 되어 탐색이 어려움
* Projects / Tasks / Detail의 관계가 직관적으로 보이지 않음

모바일에서는 **3‑panel layout을 유지하지 않고 navigation 기반 UI로 전환**한다.

즉 다음 구조로 변경한다.

```
Projects Screen
      ↓
Tasks Screen
      ↓
Task Detail Screen
```

---

# Target Mobile UX

모바일에서 사용자는 다음 흐름으로 인터페이스를 사용한다.

1. 프로젝트 목록 확인
2. 프로젝트 선택
3. 해당 프로젝트의 Task 목록 탐색
4. Task 선택
5. Task Detail 확인
6. 필요 시 Approve / Stop / Retry 수행

각 단계는 **독립된 화면**으로 구성한다.

# Layout Rules

## Mobile Breakpoints

```
>= 1200px  → Desktop layout (3 panel)
768px–1199px → Tablet layout (2 panel)
< 768px → Mobile navigation layout
```

---

# Plan Output Mobile Optimization

현재 Plan Output은 code block 때문에 horizontal scroll이 발생한다.

모바일에서는 다음 스타일을 적용한다.

```
.planOutput code {
  white-space: pre-wrap;
  word-break: break-word;
}
```

효과:

* 긴 코드 줄 자동 줄바꿈
* 좌우 스크롤 제거
* 모바일 가독성 향상

모바일에서는 다음 원칙을 적용한다.

* Status badge 대비 강화

---

# Acceptance Criteria

다음 조건을 만족해야 한다.

* 모바일에서 3‑panel UI가 사용되지 않는다
* Projects / Tasks / Task Detail이 navigation 기반으로 동작한다
* Plan Output에서 horizontal scroll이 제거된다
* Event Log는 accordion 형태로 표시된다
* Task Detail metadata는 single column으로 표시된다

---
